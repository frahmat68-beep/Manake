# Product identity
Manake is a modern media equipment rental platform for cameras, lighting, audio, HT, drones, stabilizers, and production gear.
The brand should feel:
- cinematic
- premium
- modern
- clean
- trustworthy
- practical for event organizers, students, filmmakers, and production crews

# Design direction
The UI should feel like a premium rental marketplace, not a generic admin template.
Use strong hierarchy, generous spacing, high-quality equipment visuals, refined cards, and clear CTAs.
Avoid clutter, weak contrast, generic SaaS styling, random colors, cramped layout, and inconsistent spacing.

# Homepage requirements
The homepage should include:
- a modern hero section
- a carousel or featured equipment area showing available items
- a clear explanation of what Manake is
- animated or dynamic category text such as Camera, Lighting, Audio, HT, Drone, Stabilizer
- snapshot summary of rentals: rented today, available items, upcoming bookings
- clear CTA to browse equipment
- responsive mobile layout

# Equipment card rules
Each equipment card must clearly show:
- image
- item name
- category
- availability status
- rental price if available
- CTA to view or book
Cards must be aligned, readable, and not cramped.

# UI quality checklist
Before finishing any UI task, inspect the rendered page and answer:
1. Is the first screen visually impressive within 3 seconds?
2. Is the main CTA obvious?
3. Is the layout clean and aligned?
4. Is the spacing consistent?
5. Are cards visually balanced?
6. Is the typography readable?
7. Is the contrast accessible?
8. Does it look good on desktop, tablet, and mobile?
9. Is there any overflow, broken layout, or awkward wrapping?
10. Does the page feel like a premium media equipment rental brand?

# Review guidelines
When reviewing PRs or your own changes:
- Treat broken mobile layout as a high-priority issue.
- Treat text overflow as a high-priority issue.
- Treat low contrast as a high-priority issue.
- Treat inconsistent spacing/card alignment as a high-priority issue.
- Treat missing loading/empty/error states as a high-priority issue.
- Treat ugly but functional UI as still needing improvement.
- Give concrete fixes, not vague design comments.
- Always inspect desktop, tablet, and mobile before declaring a UI task done.
- Use screenshot audits and Lighthouse as a repeatable quality gate when possible.
